# 📋 PRD (Product Requirements Document)

## 김승현 포트폴리오 포털 사이트

---

## 1. 제품 개요

### 1.1 제품명

김승현 포트폴리오 포털 사이트

### 1.2 제품 설명

다양한 게시판(공지사항, 자유 게시판, 갤러리, 문의 게시판)을 통해 사용자 간 소통과 정보 공유를 지원하는 커뮤니티형 포털 사이트. 비회원도 일부 콘텐츠 열람이 가능하며, 회원 가입 및 로그인을 통해 글 작성, 댓글, 문의 기능을 이용할 수 있다.

### 1.3 대상 사용자

- **비회원**: 사이트 콘텐츠를 열람 목적으로 방문하는 일반 방문자
- **회원(일반 사용자)**: 가입 후 글 작성, 댓글, 문의 등 참여형 기능을 이용하는 사용자
- **관리자**: 공지사항 작성, 문의 답변 및 모든 게시글 관리를 담당하는 운영자

## 2. 기술 스택 및 환경

### 2.1 기술 스택

| 구분             | 기술                          | 비고                              |
| ---------------- | ----------------------------- | --------------------------------- |
| Frontend         | Vue.js 3                      | Composition API 기반              |
| Backend          | Spring Boot 3                 | REST API 서버                     |
| Database         | MySQL 8                       | -                                 |
| 인증 방식        | JWT (sessionStorage에 저장)   | Access Token 방식, 만료 시간 적용 |
| 파일 스토리지    | 로컬 파일 시스템              | 서버 내 지정 디렉토리 저장        |
| 관리자 계정 생성 | DB 직접 삽입 (별도 가입 없음) | role 컬럼으로 관리자 구분         |

### 2.2 인증 정책

| 항목               | 정책                                                         |
| ------------------ | ------------------------------------------------------------ |
| JWT 만료 시간      | 2시간                                                        |
| 토큰 저장 위치     | sessionStorage (브라우저 탭 단위 유지, 탭 닫으면 자동 소멸) |
| 자동 갱신          | 미지원 (만료 시 재로그인 필요)                               |
| 만료 후 접근 시    | 로그인 모달 팝업 또는 `/login` 리다이렉트                    |

### 2.3 파일 업로드 정책

| 항목                 | 정책                                          |
| -------------------- | --------------------------------------------- |
| 갤러리 이미지 확장자 | `.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`      |
| 갤러리 이미지 최대   | 파일당 10MB, 게시글당 최대 10장               |

## 4. 페이지별 기능 명세

---

### 4.1 메인 페이지 (`/`)

#### 4.1.1 화면 구성

4개의 게시판 위젯이 2×2 그리드로 배치된다. 각 위젯은 게시판 최신 글 목록을 미리 보여준다.

#### 4.1.2 공지사항 위젯

- **표시 정보**: 번호 / 분류 / 제목 (최신순 6건)
- **더보기 +** 버튼: `/boards/notice` 이동
- 고정 공지(상단 핀) 게시물은 배경색 구분 표시
- 게시글 제목 클릭 시 해당 상세 페이지 이동
- **비회원/회원 동일**

#### 4.1.3 자유 게시판 위젯

- **표시 정보**: 번호 / 분류 / 제목 (댓글 수 표시, 첨부파일 아이콘 포함, 최신순 6건)
- **더보기 +** 버튼: `/boards/free` 이동
- 게시글 제목 클릭 시 해당 상세 페이지 이동
- **비회원/회원 동일**

#### 4.1.4 갤러리 위젯

- **표시 정보**: 번호 / 분류 / 썸네일 이미지 / 추가 이미지 수 (`+N`) (최신순 **4건**)
- **더보기 +** 버튼: `/boards/gallery` 이동
- **비회원/회원 동일**

#### 4.1.5 문의 게시판 위젯

- **표시 정보**: 번호 / 제목 (답변상태 및 비밀글 자물쇠 아이콘 포함)
- **더보기 +** 버튼: `/boards/inquiry` 이동
- **나의 문의 내역** 버튼
    - **비회원**: 버튼 노출되나 클릭 시 로그인 모달 팝업
    - **회원**: 클릭 시 `/boards/inquiry?my=true` 이동 → 본인 작성 글만 필터링

### 4.2 로그인 (`/login`)

#### 4.2.1 진입 방식

- GNB의 `로그인` 버튼 클릭 → `/login` 페이지 이동
- 회원 전용 기능(글 등록, 댓글 등) 클릭 시 → **로그인 모달 팝업** 표시
    - 모달에서 로그인 성공 시 현재 페이지 유지 (리다이렉트 없음)
- `/login` 직접 접근 시: 로그인 완료 후 `/` 로 이동
    - URL 패턴: `/login?ret={원래_URL}` 으로 접근 시 로그인 후 `ret` 경로로 복귀

> **정리**: `/login` 페이지와 로그인 모달은 별개로 존재한다.
> - GNB 로그인 버튼 → `/login` 페이지
> - 비회원이 회원 전용 기능 클릭 → 모달 팝업

#### 4.2.2 화면 구성

| 필드                | 설명           |
| ------------------- | -------------- |
| 아이디 (텍스트)     | 사용자 ID 입력 |
| 비밀번호 (패스워드) | 비밀번호 입력  |
| 로그인 버튼         | 제출           |
| 회원가입 링크       | `/join` 이동   |

#### 4.2.3 동작

- 로그인 성공 시: JWT를 sessionStorage에 저장, GNB가 회원 상태로 변경
- 로그인 실패 시: 인풋 하단에 `"아이디 또는 비밀번호가 올바르지 않습니다."` 메시지 표시
- 이미 로그인 상태에서 `/login`, `/join` 접근 시: `/` 로 리다이렉트

### 4.3 회원가입 (`/join`)

#### 4.3.1 화면 구성

| 필드                | 유형     | 필수 여부 | 유효성 검사 규칙                            |
| ------------------- | -------- | --------- | ------------------------------------------- |
| 아이디              | 텍스트   | 필수      | 영문+숫자 조합, 5~20자, 중복 불가           |
| 비밀번호            | 패스워드 | 필수      | 영문+숫자 조합, 최소 8자 이상               |
| 비밀번호 확인       | 패스워드 | 필수      | 비밀번호와 일치 여부 확인                   |
| 이름                | 텍스트   | 필수      | 1~20자                                      |
| 회원 가입 하기 버튼 | 제출     | -         | -                                           |

- 모든 필드 입력 완료 전까지 버튼 비활성화(회색) 처리
- 아이디 중복 확인: 폼 **제출 시** 서버에서 검사 (실시간 체크 미지원)
- 중복 아이디 제출 시: `"이미 사용 중인 아이디입니다."` 메시지 표시
- 로그인 상태에서 접근 시: `/` 로 리다이렉트

- URL: `/boards/notice/modify/{id}`

## 7. 비회원 vs 회원 경험 비교 요약

### 비회원 접근 시나리오

비회원은 사이트의 모든 콘텐츠를 **읽기 전용**으로 이용할 수 있다. 단, 글 작성, 댓글 작성, 비밀 문의글 열람, 나의 문의 내역 확인은 불가하며, 이러한 기능을 시도할 경우 로그인 모달이 팝업되거나 동작이 차단된다. 자유 게시판 상세 페이지에서 댓글 입력란은 시각적으로 노출되지만, 등록 시도 시 로그인을 요구한다.

### 회원 접근 시나리오

로그인한 회원은 GNB에서 이름과 로그아웃 버튼을 확인할 수 있다. 모든 게시판에서 글을 작성할 수 있고, 댓글을 등록할 수 있으며, 본인이 작성한 글과 댓글을 수정·삭제할 수 있다. 문의 게시판에서는 비밀글 작성 기능을 통해 개인적인 문의를 남길 수 있으며, 나의 문의 내역 보기 기능으로 본인 글만 필터링해서 볼 수 있다. 단, 답변이 완료된 문의글은 수정할 수 없다.

## 8. URL 파라미터 명세

게시판 목록 페이지에서 사용되는 쿼리 파라미터:

| 파라미터         | 설명                        | 예시                              |
| ---------------- | --------------------------- | --------------------------------- |
| `startDate`      | 검색 시작일                 | `2025-04-11`                      |
| `endDate`        | 검색 종료일                 | `2026-04-11`                      |
| `category`       | 카테고리 코드 (`-1` = 전체) | `-1`, `1`, `2`                    |
| `searchText`     | 검색 키워드 (최소 2자)      | `안녕`                            |
| `pageSize`       | 페이지당 표시 건수          | `10`, `20`, `30`                  |
| `orderValue`     | 정렬 기준                   | `createdAt`, `title`, `viewCount` |
| `orderDirection` | 정렬 방향                   | `desc`, `asc`                     |
| `pageNum`        | 현재 페이지 번호            | `1`, `2`                          |
| `my` (문의만)    | 나의 문의 내역 필터         | `true`, `false`                   |
