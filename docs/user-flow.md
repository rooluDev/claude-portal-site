## 공통 사전 조건 정의

| 용어         | 설명                                                                 |
| ------------ | -------------------------------------------------------------------- |
| 비회원       | sessionStorage에 JWT가 없는 상태                                     |
| 회원         | sessionStorage에 유효한 JWT가 있고, role = `USER`                    |
| 관리자       | sessionStorage에 유효한 JWT가 있고, role = `ADMIN`                   |
| 토큰 만료    | JWT가 있으나 만료 시간(2시간) 초과 → 비회원과 동일하게 처리         |
| 로그인 모달  | 현재 페이지를 유지한 채 overlay로 띄우는 로그인 폼                   |

## UF-001. 회원가입

**진입점**: GNB `회원가입` 링크 클릭 → `/join`

```
1. 사용자가 /join 페이지 진입
   ├─ [로그인 상태] → / 로 리다이렉트 (종료)
   └─ [비로그인 상태] → 회원가입 폼 표시

2. 사용자가 아이디 / 비밀번호 / 비밀번호 확인 / 이름 입력
   ├─ [모든 필드 미입력 또는 일부 공백] → 가입 버튼 비활성화 (회색)
   └─ [모든 필드 입력 완료] → 가입 버튼 활성화

3. 사용자가 '회원 가입 하기' 버튼 클릭

4. 클라이언트 유효성 검사
   ├─ 아이디: 영문+숫자, 5~20자 위반 → 해당 필드 하단 에러 메시지 표시 (종료)
   ├─ 비밀번호: 영문+숫자, 8자 이상 위반 → 해당 필드 하단 에러 메시지 표시 (종료)
   ├─ 비밀번호 확인: 불일치 → 해당 필드 하단 에러 메시지 표시 (종료)
   └─ [모든 검사 통과] → 서버로 가입 요청 전송

5. 서버 응답
   ├─ [아이디 중복] → "이미 사용 중인 아이디입니다." 메시지 표시 (2단계로 복귀)
   ├─ [서버 오류] → "잠시 후 다시 시도해주세요." 메시지 표시 (종료)
   └─ [가입 성공] → /login 페이지로 이동 (완료)
```

## UF-002. 로그인 (페이지)

**진입점**: GNB `로그인` 버튼 클릭 → `/login`

```
1. 사용자가 /login 페이지 진입
   ├─ [로그인 상태] → / 로 리다이렉트 (종료)
   └─ [비로그인 상태] → 로그인 폼 표시

2. 사용자가 아이디 / 비밀번호 입력 후 '로그인' 버튼 클릭

3. 서버 응답
   ├─ [실패] → "아이디 또는 비밀번호가 올바르지 않습니다." 메시지 표시 (2단계로 복귀)
   └─ [성공] → JWT를 sessionStorage에 저장, GNB 회원 상태로 갱신
               ├─ [ret 파라미터 있음] → ret 경로로 이동 (완료)
               └─ [ret 파라미터 없음] → / 로 이동 (완료)
```

## UF-003. 로그인 (모달 팝업)

**진입점**: 비회원이 회원 전용 기능(글 등록, 댓글 등록, 나의 문의 등) 클릭

```
1. 비회원이 회원 전용 액션 클릭
   → 현재 페이지 위에 로그인 모달 팝업 표시

2. 사용자가 아이디 / 비밀번호 입력 후 '로그인' 버튼 클릭
   ├─ [실패] → 모달 내 "아이디 또는 비밀번호가 올바르지 않습니다." 표시 (2단계로 복귀)
   └─ [성공] → JWT 저장, GNB 회원 상태로 갱신, 모달 닫기
               → 현재 페이지 유지 (페이지 이동 없음) (완료)

3. 사용자가 '회원가입' 링크 클릭
   → 모달 닫기 → /join 으로 이동

4. 사용자가 배경 클릭 또는 ESC 키 입력
   → 모달 닫기, 현재 페이지 유지 (종료)
```

## UF-004. 로그아웃

**진입점**: GNB `로그아웃` 버튼 클릭 (회원/관리자)

```

2. sessionStorage에서 JWT 삭제

3. GNB 비회원 상태로 갱신

4. / 로 이동 (완료)
```

## UF-005. JWT 만료 처리

**진입점**: 로그인 후 2시간 초과 상태에서 회원 전용 기능 접근

```
1. 사용자가 회원 전용 API 요청 발생
   (글 등록, 수정, 삭제, 댓글 등록 등)

2. 서버가 401 Unauthorized 응답

3. 클라이언트 처리
   ├─ sessionStorage에서 JWT 삭제
   ├─ GNB 비회원 상태로 갱신
   └─ [페이지 내 액션이었을 경우] → 로그인 모달 팝업 표시
      [페이지 직접 접근이었을 경우] → /login?ret={현재URL} 으로 리다이렉트
```

## UF-010. 메인 페이지 탐색

**진입점**: 브라우저에서 `/` 접근 또는 GNB 로고 클릭

```
1. 메인 페이지 진입
   → 4개 위젯(공지사항 / 자유게시판 / 갤러리 / 문의게시판) 렌더링

2-A. 공지사항 위젯
   ├─ 게시글 제목 클릭 → /boards/notice/{id} 로 이동
   └─ '더보기 +' 클릭 → /boards/notice 로 이동

2-B. 자유게시판 위젯
   ├─ 게시글 제목 클릭 → /boards/free/{id} 로 이동
   └─ '더보기 +' 클릭 → /boards/free 로 이동

2-C. 갤러리 위젯
   ├─ 게시글 카드 클릭 → /boards/gallery/{id} 로 이동
   └─ '더보기 +' 클릭 → /boards/gallery 로 이동

2-D. 문의게시판 위젯
   ├─ 게시글 제목 클릭
   │   ├─ [비밀글 + 비회원] → 로그인 모달 팝업
   │   ├─ [비밀글 + 타인 회원] → 토스트 메시지 "권한 없음"
   │   └─ [일반글 또는 본인/관리자] → /boards/inquiry/{id} 로 이동
   ├─ '더보기 +' 클릭 → /boards/inquiry 로 이동
   └─ '나의 문의 내역' 버튼 클릭
       ├─ [비회원] → 로그인 모달 팝업
       └─ [회원] → /boards/inquiry?my=true 로 이동
```

## UF-020. 공지사항 목록 조회 / 검색

**진입점**: GNB `공지사항` 클릭 또는 `/boards/notice`

```
1. /boards/notice 진입
   → 고정 공지(회색 배경) + 일반 게시글 목록 표시
   → 기본 정렬: 등록일시 내림차순, 기간: 최근 1년, 페이지당 10건

2. [선택] 검색 / 필터 조작
   ├─ 기간 필터 변경 (Date Picker)
   ├─ 카테고리 변경 (전체 / 공지 / 이벤트)
   ├─ 키워드 입력 (최소 2자)
   ├─ 개씩 보기 변경 (10 / 20 / 30)
   ├─ 정렬 기준 / 방향 변경
   └─ '검색' 버튼 클릭 → URL 파라미터 갱신 후 목록 재조회

3. [선택] 페이지네이션 숫자 클릭 → 해당 페이지로 이동

4. 게시글 제목 클릭 → UF-021 (상세 조회) 진행
```

## UF-021. 공지사항 상세 조회

**진입점**: 공지사항 목록에서 제목 클릭 → `/boards/notice/{id}`

```
1. /boards/notice/{id} 진입
   → 게시글 상세 표시 (분류 태그 / 제목 / 작성일 / 작성자 / 조회수 / 본문)
   → 동일 세션 첫 방문이면 조회수 +1

2. 버튼 표시 분기
   ├─ [비회원 / 일반 회원] → '목록' 버튼만 표시
   └─ [관리자] → '목록' / '수정' / '삭제' 버튼 표시

3-A. '목록' 클릭 → /boards/notice 로 이동

3-B. '수정' 클릭 (관리자)
   → /boards/notice/modify/{id} 로 이동 (UF-023)

3-C. '삭제' 클릭 (관리자)
   → UF-024 진행
```

## UF-022. 공지사항 글 작성

**진입점**: `/boards/notice` 에서 '글 등록' 버튼 클릭 (관리자만 노출)

```
1. 관리자가 '글 등록' 클릭 → /boards/notice/write 진입

2. 폼 작성
   ├─ 카테고리 선택 (공지 / 이벤트, 필수)
   ├─ 고정 공지 체크박스 선택 (선택)
   ├─ 제목 입력 (필수)
   └─ 내용 입력 (필수)

3. '취소' 클릭 → /boards/notice 로 이동 (종료)

4. '등록' 클릭
   ├─ [필수 항목 미입력] → 해당 필드 에러 메시지 표시 (2단계로 복귀)
   └─ [서버 등록 성공] → 새로 작성된 게시글 상세 페이지로 이동 (완료)
```

## UF-031. 자유게시판 상세 조회

**진입점**: 자유게시판 목록에서 제목 클릭 → `/boards/free/{id}`

```
1. /boards/free/{id} 진입
   → 게시글 상세 표시 (분류 태그 / 제목 / 작성일 / 작성자 / 조회수 / 본문 / 첨부파일 목록)
   → 동일 세션 첫 방문이면 조회수 +1

2. 버튼 표시 분기
   ├─ [비회원] → '목록' 버튼만 표시
   ├─ [회원(타인)] → '목록' 버튼만 표시
   ├─ [회원(본인)] → '목록' / '수정' / '삭제' 버튼 표시
   └─ [관리자] → '목록' / '수정' / '삭제' 버튼 표시 (모든 글에 대해)

3-A. '목록' 클릭 → /boards/free 이동

3-B. '수정' 클릭 (본인 / 관리자) → /boards/free/modify/{id} 이동 (UF-033)

3-C. '삭제' 클릭 (본인 / 관리자) → UF-034 진행

4. 댓글 영역
   → 댓글 목록 표시 (비회원 포함 전체 열람 가능)
   → 댓글 입력란 항상 노출
   ├─ 댓글 등록 시도 (UF-035)
   └─ 본인 댓글 삭제 시도 (UF-036)
```

## UF-032. 자유게시판 글 작성

## UF-057. 관리자 답변 수정 / 삭제

**진입점**: 문의게시판 상세 (답변완료 상태) - 관리자만 노출

```
[답변 수정]
1. 관리자가 '답변 수정' 버튼 클릭
   → 기존 답변 내용이 수정 가능한 텍스트 입력란으로 전환

2. 내용 수정 후 '저장' 클릭
   ├─ [내용 비어있을 경우] → 에러 메시지 (2단계로 복귀)
   └─ [성공] → 답변 내용 갱신, 읽기 상태로 전환 (완료)

3. '취소' 클릭 → 수정 취소, 기존 답변 표시 유지 (종료)

[답변 삭제]
1. 관리자가 '답변 삭제' 버튼 클릭
   → confirm 다이얼로그: "답변을 삭제하시겠습니까?"

2. [취소] → 다이얼로그 닫기 (종료)

3. [확인] → 서버 답변 삭제 요청
   └─ [성공] → 페이지 갱신
               - 문의 상태 '답변완료' → '미답변' 으로 복귀
               - 답변 영역에 입력란 + '답변 등록' 버튼으로 전환 (완료)
```
